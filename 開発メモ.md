# NEON SURVIVOR - 開発ナレッジ・実装メモ

本ドキュメントは、ゲームシステム開発における設計、各機能の仕様、およびバグ防止のための注意点をまとめた開発メモです。トークン節約や後続の開発者への引き継ぎに役立ててください。

---

## 1. 基本設計・アーキテクチャ

* **テクノロジースタック**: HTML5, Canvas, JavaScript (Vanilla ES6), CSS。外部ライブラリを使わず軽量で高速に動作します。
* **描画解像度（Canvas）**: 論理解像度 `900x600`（3:2アスペクト比）の固定値。画面サイズに合わせて自動スケーリングされます。
* **プレイヤー座標制御**: プレイヤーの位置 (`x`, `y`) は Canvas の境界内にクランプ（制限）されます。
* **サウンドシステム ([audio.js](file:///c:/Users/vb478/Documents/antigravity/vanpaia_game/audio.js))**: Web Audio API を用いて効果音をリアルタイムに波形合成 (`AudioSynthManager`)。音声ファイルのロードが不要で、CORSの影響も受けません。

---

## 2. 武器パラメータバランスエディタ & 外部設定化

* **データ読み込みフロー**:
  1. ゲーム起動時に `weapon_balance.json` を非同期で `fetch`。
  2. **CORS / ローカル実行対策**: `file://` プロトコルによるローカル起動等で fetch がブロックされた場合、[entities.js](file:///c:/Users/vb478/Documents/antigravity/vanpaia_game/entities.js) 内のメモリ上のデフォルト設定オブジェクトにフォールバックします。
* **パラメータ調整ダッシュボード ([balance_editor.html](file:///c:/Users/vb478/Documents/antigravity/vanpaia_game/balance_editor.html))**:
  * 各武器（1〜10レベル）の数値（クールダウン、攻撃範囲、威力等）をスライダーで直感的に編集可能。
  * JSONのインポート（読み込み）およびエクスポート（ローカル保存）に対応。
* **武器威力・パラメータ計算公式**:
  レベルごと個別の値がないパラメータについて、ベース値とレベルごとの成長率から一括計算できるよう統一：
  $$\text{適用値} = \text{基本値} \times (1 + \text{成長率} \times (\text{現在のLv} - 1)) \times \text{補正値}$$

## 3. スポーン制御 & 進行バグ対策（重要）

### ⚠️ 配列のフィルター上書きバグ（JavaScriptの落とし穴）
* **不具合内容**: `resolveCollisions()` 内で `this.enemies = this.enemies.filter(enemy => { ... })` によって死亡した敵の除外を行っている最中に、ボス1の撃破を検知して `this.spawnBoss2()`（内部で `this.enemies.push(boss2)`）を呼び出すと、新しく追加された `boss2` は**フィルター処理後の配列代入によって上書きされ、消滅してしまう不具合**がありました。
* **対策**: `spawnBoss2()` は配列へ直接プッシュせず生成したインスタンスを戻り値として返し、フィルターの外側で退避させておいた一時配列（`newSpawns`）に格納。フィルター終了後に `this.enemies = this.enemies.concat(newSpawns)` で結合することで安全にスポーンさせます。

### ⚙️ スポーン距離と過密化の解消
* ボスの出現座標をプレイヤーから `450px`（画面外）から **`200px`（画面内）** に変更し、出現時にすぐ視認できるようにしました。
* ボス生存中は、敵スポーンのタイマー処理をフリーズさせ、雑魚敵が画面を埋め尽くして処理落ちや視認性の低下を招くのを防ぎます。
* デベロッパーパネルの「CLEAR ENEMIES」ボタンは `this.enemies = []` でクリアするのではなく、すべての敵に `enemy.takeDamage(Infinity)` を適用して、撃破トリガー（次ボスの出現やクリア判定）をスキップせず正常に作動させます。

## 4. 難易度設定（HARDモード）

* **選択方法**: スタート画面の `NORMAL START` (サイアン) / `HARD START` (ピンク) ボタンで難易度ステータスを制御。
* **仕様**: `this.difficulty === 'HARD'` の場合、すべての敵（雑魚、分裂スライム、ボス1&2）の生成時HPスケール倍率を**自動的に2.0倍**にします。
* **UI**: プレイ中はHUDのタイマー横に `[HARD]` のピンクネオンバッジが表示され、リザルト画面の「難易度」行に記録が出力されます。

## 5. プレイヤー蘇生システム (Revival)

* **蘇生確認（暗転一時停止）**:
  * 体力が0になった際、残り蘇生回数がある場合は `this.state = 'REVIVING'` へ遷移してゲームループを一時停止。
  * 半透明の暗黒オーバーレイ (`#revive-screen`) に残り回数を表示し、「復活する」ボタンを押すと再開します。
* **復活エフェクト (VFX)**:
  * **画面全体フラッシュ**: 復活時に画面全体を白青 (`rgba(255, 255, 255)`) でフラッシュさせ、600msかけてフェードアウト。
  * **画面全体の衝撃波**: プレイヤーの位置を中心に、円形のネオンサイアンリングが最大半径 `650px` まで高速で拡大。距離に応じて線幅と透明度が減衰する滑らかなフェードアウト処理。
  * **ノックバック**: 周囲250px以内の敵を80px外側へ吹き飛ばし、2秒間の無敵（I-frame）とHP全回復を与えます。

## 6. 背水バフ（低HP時の赤いオーラ）

* **発動条件**: プレイヤーのHP比率が最大HPの20%未満 (`hp / maxHp < 0.20`) のときに自動発動。
* **ビジュアルエフェクト**: 画面の端を覆うように、ゆっくりと明滅（パルス）する赤い放射状グラデーション（ヴィニエット）と、外周の赤いネオン境界線を表示。
* **バフ効果**: 攻撃力が**1.5倍**になります。[entities.js](file:///c:/Users/vb478/Documents/antigravity/vanpaia_game/entities.js) 内の `Player` クラス constructor にて `stats.damageMultiplier` プロパティを Getter / Setter を使ってカプセル化することで、HPが20%を下回ったときのみ動的に1.5倍で算出。武器側のロジックを変更することなく、バフの自動付与と解除を実現しています。

## 7. 雑魚敵の画像出現 (Hutty)

* **仕様**: 雑魚敵（ボス以外）のスポーン時に、**20%の確率**でベクター多角形ではなく `img/hutty.png` 画像をレンダリングします。
* **アセット**: 画像オブジェクトを `entities.js` 最下部で `new Image()` として1回だけ静的に事前ロードし、描画のオーバーヘッドを削減。
* **サイズと挙動**: 敵の `radius * 2.5` にサイズを同期させており、敵の種類（ゴーレム＝大、コウモリ＝小）に応じてサイズが変わります。足踏みアニメーションや被弾時の揺れも画像に適用され、被弾時は白い glowing 枠線が点滅します。

## 8. 宝石ドロップ & ルーレット抽選

* **ドロップ確率**: 雑魚敵撃破時に **5%の確率** で出現。通常のジェムより一回り大きく、金色に輝く8つ角の星マーク（`Jewel` クラス）として描画されます。
* **アップグレード数選択**: 回収時にゲームを一時停止 (`state = 'ROULETTE'`) し、**1〜3種類**のパワーアップ個数を抽選。
* **ルーレット演出（SE & VFX）**:
  * 全武器・パッシブのアイコンを高速シャッフルし、Web Audio API による高音の「カタカタ」としたスロット音 (`playRouletteTick()`) を再生しながら徐々に減速。
  * 確定時にレベルアップファンファーレが鳴り、対象武器がアップグレード。
  * 確定の瞬間に、ルーレット画面の背景が**サイアン色に一瞬フラッシュ**し、同時にボックス中央から**40発のカラフルなネオンHTMLパーティクルが爆発的に飛び散る**演出を行います。
  * 抽選中は「閉じる」ボタンがロックされ、すべての抽選結果がリストアップされるとアクティブ化してゲームに戻れます。
